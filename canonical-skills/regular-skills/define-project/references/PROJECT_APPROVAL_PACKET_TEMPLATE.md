# Project Approval Packet

## Project Definition Artifacts

List the exact project-definition files and versions or hashes being presented.

## Decision Readiness Evidence

Reference the Decision Readiness Summary and confirm whether `Blocking Open Decisions` is `None`.

## Walking Skeleton

State the executable end-to-end outcome and the principal boundaries it crosses.

## Roadmap Outcomes

Summarize each phase by observable outcome and dependency order.

## Intentionally Deferred Decisions

List each deferred decision with:

- why deferral is safe now;
- affected scope;
- decision owner;
- the Roadmap Decision Gate or other blocking trigger;
- when it becomes blocking.

## Risks and Non-Blocking Concerns

Separate known risks from assumptions and unresolved but non-blocking concerns.

## Approval Requested

Ask the human to approve or reject the exact Project Definition. State that approval authorizes eligibility for a separately invoked `bootstrap-project`; it does not authorize production implementation, dependency changes, migrations, deployment, Git operations, or automatic workflow invocation.
